Albert and the ear shells¶

Ear shells are marine snails

The age of ear shells can be determined by cutting the shell through the cone, staining it, and counting the number of rings through a microscope - a boring and time-consuming task.

Albert, a scientist and ear shells specialist, needs your help. He has collected data on his favorite sea creature and asks you to help him understand it. He is looking for your help to develop a model to predict the number of rings of ear shells (a proxy to their age) using other measurements which are easier to obtain.

What I see here is that the four weight-related features show the strongest correlation with price. All exhibit a clear positive linear relationship — heavier ear shells are more expensive. Females (red) consistently command higher prices than males (blue) for similar weights, while infants (green) cluster at the lower end. weight_whole appears to be the single best predictor of price. This suggests price is primarily driven by size/weight, with sex playing a secondary role (likely because females tend to grow larger).

1: Linear Regression — Simple, interpretable, fast to train. Albert can easily understand which features matter and by how much (coefficients). However, it assumes a linear relationship and may underfit if the true relationship is more complex.

2: Random Forest Regressor — Captures non-linear relationships and feature interactions automatically. More robust to outliers and doesn't require feature scaling. However, it's a "black box" and harder to interpret.

My choice: Random Forest — While interpretability is valuable for Albert, accurately predicting ring count is the primary goal. Random Forest typically outperforms linear models on tabular data with potential non-linear patterns, and we can still use feature importances to provide some interpretability.

# Prepare data using the pre-assigned train/test split
train = df[df['subset'] == 'train'].copy()
test = df[df['subset'] == 'test'].copy()

# Encode 'sex' column
le = LabelEncoder()
train['sex_encoded'] = le.fit_transform(train['sex'])
test['sex_encoded'] = le.transform(test['sex'])

# Features (exclude subset, sex, rings, price)
features = ['sex_encoded', 'length', 'diameter', 'height', 
            'weight_whole', 'weight_shucked', 'weight_viscera', 'weight_shell']

# Drop rows with NaN in rings
train = train.dropna(subset=['rings'])
test = test.dropna(subset=['rings'])

X_train = train[features]
y_train = train['rings']
X_test = test[features]
y_test = test['rings']

print(f"Train size: {X_train.shape[0]}, Test size: {X_test.shape[0]}")

Performance Interpretation:

MAE of 0.886 means on average our prediction is off by less than 1 ring, which is reasonable given that ring count is an integer. R² of 0.55 means the model explains about 55% of the variance in ring count — decent but with room for improvement. The Actual vs Predicted plot shows the model performs well for mid-range values but struggles with extreme values (very young or very old shells). Limitations:

The model only explains ~55% of the variance, suggesting important predictive information may be missing from the dataset. Random Forest can overfit on small datasets — cross-validation would give a more robust estimate. The sex encoding (LabelEncoder) introduces an artificial ordinal relationship between categories. One-hot encoding would be more appropriate. Potential improvements:

Maybe useing XGBoost as well and compare the result Use one-hot encoding for the sex feature instead of label encoding. Hyperparameter tuning (max_depth, min_samples_split) via grid search or cross-validation. Feature engineering: create interaction features (e.g., weight ratios, volume estimates).
